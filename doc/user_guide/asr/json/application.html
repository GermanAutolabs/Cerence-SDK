Application Configuration Options

OVERVIEW

An application contains the ASR/Cloud/NLU configuration for a given ASR/Cloud/NLU task. That is, it can configure a set of search, queries and possibly other things that are used when an utterance is processed by Cerence ASR. In this document, we provide a brief overview of application types and their options.

COMMON OPTIONS

Every application is build up following this schema:

{
  "version" : "VoConHigh 5.0",
  "application" : [{
    "name" : "application_name",
    "input": [{
      "name" : "input_name",
      "type" : "<typeName>",
      "<typeName>" : {
        ...
      }
    }],
    "output": [{
      "name" : "output_name",
      "type" : "<typeName>",
      "<typeName>" : {
        ...
      }
    }],
    "nlu": [{
      "name" : "nlu_name",
      "type" : "<typeName>",
      "<typeName>" : {
        ...
      }
    }]
  }]
}
name
Uniquely identifies the input.
input
configure for recognition, the type can be can be asr and cloud at the moment.
output
configure for recognition, the type can be can be internalAudio at the moment.
nlu
configure for nlu, the can be sem3 only.

ASR INPUT

The asr input can be used to configure almost any recognition task. This includes tasks as complex as the all-in-one menu (A1M). The following configuration is a reduced version of said A1M. It gives an example of all available configuration options for the asr input:

{
  "version" : "VoConHigh 5.0",
  "input" : [{
    "name" : "APP_CNC",
    "type" : "asr",
    "asr" : {
      "search" : [{
          "name" : "CNC_CTX",
          "type" : "static",
          "static" : {
            "file_name" : "modelm_small.fcf"
          }
        },{
            "name":"VAD_NAMES",
            "type" : "dynamic",
            "dynamic" : {
                "dynamic_content_consumer" : "DCC_LOCAL_VAD_NAMES",
                "search_name" : "vad_lastname.fcf"
            }
        }

      ],
      "dccs_to_wait_for" : [{
              "name" : "DCC_LOCAL_VAD_NAMES"
          }
      ]
    }
  }]
}

The asr object contains the following keys.

search

A list of search objects to be used in recognition. Each search object can contain the following keys.

name
Unique name of the search. Used in the result.
type
Is either static or dynamic
static

A static context (found on disk)

filename
Filename of the context. This path is relative to the search path specified in the `asr_paths configuration`_'s ``asr_paths
file_path
The absolute path to the context. This is mutually exclusive with filename.
dynamic

A context generated online by a DCC

dynamic_content_consumer
The name of the DCC
search_name
The name (not filename) of the context generated by the DCC
source
The name of the source that identifies the source (e.g. device name) of the dynamic data.
vocon_parameters
ASR parameters related to search spaces. Possible parameters are described in the vocon parameters documentation. Parameter values specified here take precedence over values specified in the recognizer configuration for the time of recognition of the current use-case.
consolidated_update

Configures consolidated update for search spaces that have consolidated update buffer.

filename

The consolidated update file name. It is a required field in case the file_path key is not provided. The file should be located under context_update asr_paths configuration’s asr_paths in this case.

search path specified in the `asr_paths configuration`_'s ``asr_paths

file_path
The consolidated update absolute path (including the filename). It is a required field in case the filename key is not provided
dccs_to_wait_for

List of dynamic content consumers that the recognition will wait for before starting. This key is optional. Its values can be changed via vh_callback. If not specified, the recognition will start with the data available at that point. All DCCs listed here must be defined in the same application configuration. Otherwise, they will be ignored and a warning will be logged. Each object contains the following keys:

name
The name of the DCC to wait for.
source
The source of the DCC to wait for. This key is optional.

CLOUD INPUT

The following configuration gives an example of all available configuration options for the cloud input:

{
  "version": "VoConHigh 5.0",
  "input": [{
    "name" : "MY_CLOUD_APP",
    "type" : "cloud",
    "cloud" : {
      "cloud_services_name" : "csHttp",
      "query" : {
        "name" : "REMOTE_QUERY_HTTP",
        "command" : "DRAGON_NLU_ASR_CMD",
        "connection_info" : {
          "uses" : "DRAGON_CONNECTION_INFO_HTTP"
        },
        "param" : {
          "type" : "REQUEST_INFO",
          "REQUEST_INFO" : {
            "json" : {
              "start" : 0,
              "end" : 0,
              "text" : "",
              "appserver_data" : {
                "message" : "SLOTS:GLOBAL:reset_cancel",
                "timezone" : "America\\/Toronto",
                "speaker" : "user",
                "error_on_user_data_not_uploaded" : "0",
                "type" : "conversation",
                "actions" : []
              },
              "binary_results" : 0
            }
          }
        }
      }
    }
  }]
}

The cloud object has the following keys.

cloud_services_name
The name of the cloud services to use for connection to the server.
query

The query object to use with the connection.

name
The name of the query.
command
The command to send to the server.
connection_info
The connection info to send as part of the query. This is a verbatim json object.
param

Paramters for query.

type
Type of this set of parameters. Can currently only be REQUEST_INFO.
REQUEST_INFO
The REQUEST_INFO parameter object. This is a verbatim json object.

SEM3 NLU

The following configuration gives an example of all available configuration options for the sem3 input:

{
  "version" : "VoConHigh 5.0",
  "nlu" : [{
    "name" : "SEM3_MESSAGING",
    "type" : "sem3",
    "sem3" : {
      "app": {
        "name": "MUSIC_APP",
        "file_name": "itn_enu_messaging_v2_0_v5.s3c"
      }
    }
  }]
}

The app object contains the following keys.

app

name
Unique name of the sem3 app. Used in the result.
file_name
The name on disc of the sem3 app to load. It is a required field in case the file_path key is not provided.
filePath
The absolute path (including the filename) of the sem3 app to load.
additionalData

additional data to a particular annotator that can have multiple data receivers.

path
A data path is a combination of a unique annotator identifier, optionally with one or more data receiver identifier.
configuration
Specify configuration for a SEM3 context that have multiple data receivers.
data
Sem3 data, for now, only static data supported.
filter
The filter applied to sem3 additional data.

internalAudio OUTPUT

The internalAudio OUTPUT can be used to route the audio received by Cerence ASR to other audio module like Prompter, Biometry, AudioToFile etc. The configuration take effect when setApplication interface is called, the audioStartTimeMs, audioEndTimeMs define the section of the audio interested. An application json may contain only “output” section, no asr/cloud input and sem3 nlu, in this case, the only purpose of the application is to routing audio. It gives an example of all available configuration options for the internalAudio output:

{
  "version" : "VoConHigh 5.0",
  "output" : [{
    "name" : "to_file",
    "type" : "internalAudio",
    "internalAudio" : {
       json string,
    }
  }, {
    "name" : "to_prompter",
    "type" : "internalAudio",
    "internalAudio" : {
       json string,
    }
  }]
}
