VERBATIM JSON CONFIGURATION OPTIONS

OVERVIEW

Verbatim JSON objects allow adding arbitrary JSON as a named object that can be resolved by the Cerence ASR configuration.

OPTIONS

Generally, a verbatim JSON object has the following fowm:

"<key>": {
  "name": "<somename>",
  "json": {
    ...
  }
}

It consists of a name and a json key and value. The name is optional and can be used to identify the object, e.g. for reference by a uses directive. The value of the json key has to be an object. It is used as is, i.e. no further validation is done by the Cerence ASR engine, except for resolving callback definitions. This means that every value of every key or sub-key in the value of the json key can be a vh_callback (see example below).

For example, the credentials object uses verbatim json that is passed to Cerence ASR internally (and then to the server):

"credentials": {
  "name": "MyCredentials",
  "type": "http",
  "http": {
    "url": "http://example.com",
    "json": {
      "appId": "MY_APP_ID",
      "appKey": "1234567890abcdef",
      "uId": "id",
      "inCodec": "PCM_16_16K",
      "outCodec": "PCM_16_16K"
    }
  }
}

Similarly, the connection_info object also uses verbatim json. The following shows an example of a nested vh_callback definition as well:

"connection_info" : {
  "name" : "MyConnection",
  "type": "http",
  "http": {
    "json" : {
      "appName" : "vocon_high_rec_http",
      ...,
      "cmdDict" : {
        "dictation_type" : "Dictation",
        ...,
        "application_state_id" : {
          "vh_callback" : {
            "name" : "application_state_id",
            "type" : "string",
            "default" : "45"
          }
        }
      }
    }
  }
}

The callbacks will all be resolved before passing the json block to its users.
