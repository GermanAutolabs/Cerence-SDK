Introduction

This Software Development Kit (SDK) provides access to Cerence’s state of the art speech technology software through high-level application programming interfaces (API) that can be used for developing embedded and hybrid speech applications.

The following components are included:

  • ASR: high-level API framework layer providing access to both Embedded ASR and Cloud ASR on Cerence Cloud Services
  • Prompter: high-level API framework layer providing access to both Cerence TTS Embedded and Cerence Cloud TTS on Cerence Cloud Services
  • Audio: audio framework providing high-level audio management for all audio processing modules
  • Common: General functionality needed by all components, e.g. processing of JSON configurations, logging
  • CloudServices: component that provides connection with Cerence Cloud Services, used by ASR and Prompter components

Application developers working with the SDK have the flexibility to use either single components or combine them into complex speech dialog applications. Each component comes with its own documentation and sample programs which will be a good starting point for your own implementations.

The SDK is available for the following standard build platforms (see readme files of platform-specific packages for build details):

  • Android
  • iOS
  • Linux
  • Windows

Porting to other platforms can be offered by Cerence Professional Services if the target system meets the CSDK Porting Requirements.

SDK packages for the single platforms have differences with respect to content and structure. This document describes the common concepts of all packages, platform-specific details are described in additional documents in the single packages.

Samples

The SDK contains sample programs as source code that demonstrate how to use the API.

Note that the main purpose of the sample programs and the included sample data is to illustrate how speech applications can be implemented using the APIs of the SDK. The sample code and the use case configurations are not meant as best-practice recommendations regarding voice user interface design and implementation. They are provided as examples that can be used as starting point for the implementation of your own applications.

Language specific data for the samples are included for testing purposes in American English (ENU). Other languages are available on request as separate ASR and TTS language packages.

Details how to compile and run the samples can be found in the additional documentation of the platform-specific SDK packages.

Logging

All SDK components provide a logger which can be configured flexibly with a JSON configuration file. See Common User’s Guide.

JSON Configuration

All SDK components need to be configured with JSON (JavaScript Object Notation) files. Each component has its own specific set of JSON configuration parameters which are described within the component-specific user guides.

JSOn configuration files need to read in by the application using the common_IConfiguration API described in the document Common User’s Guide.

Threading Model of SDK Components

SDK components are multithreaded. All threads that are created by SDK components are described in the document Common User’s Guide.

Note that thread priorities need to be configured properly by system integrators to ensure a good performance on the target system.

Free and Open Source Software (FOSS)

SDK components rely on particular open source software packages. The copyright and licensing information for these packages are included in the document SDK Open-Source.
