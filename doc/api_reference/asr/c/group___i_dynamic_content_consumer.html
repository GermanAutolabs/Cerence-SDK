typedef struct _nuance_asr5_IDynamicContentConsumer nuance_asr5_IDynamicContentConsumer

Provides a consumer for complex, connected, and dynamic data used for ASR.

The DynamicContentConsumer can be used to

  • load a precompiled database (e.g. with POIs)
  • upload data to a remote server
  • hold dynamic vocabulary entries, and create transcriptions using an automatic grapheme to phoneme engine (e.g. clc)

nuance_asr5_IContentValuesIterator *  iterator

Start data preperation.

Starts data preparation in a worker thread. Each dynamic content consumer uses a separate worker thread.

All applications that use data from this dynamic content consumer will only finish to be created when the dynamic content consumer is finished preparing its data. This synchronization happens inside vocon high, there is no need to explicitly call IDynamicContentConsumer::waitForCompletion().

Keep in mind that the online grapheme to phoneme conversion might take a lot of time to complete, especially on a large number of entries.

Parameters
[in,out]instanceThe instance pointer
[in]iteratorA set of callbacks that vocon high uses to retrieve all content.
