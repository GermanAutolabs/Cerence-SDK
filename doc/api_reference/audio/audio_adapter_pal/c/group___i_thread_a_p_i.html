typedef struct _nuance_common_IThreadAPI nuance_common_IThreadAPI

Provides interface for the customer to implement and install external thread management functions.

This structure is the interface to the framework class that implements API for external thread management.

typedef unsigned long posix_t

Thread id type for platforms using POSIX pthread lib

typedef void* windows_t

Thread id type for Windows platforms

typedef int(* nuance_common_IThreadAPI_CreateFunc) (void *pIntFunc, void *pIntThreadData, nuance_common_IThreadAPI_External *pExtThreadData)

User defined external function that creates new thread according to the parameters defined by the framework.

External function is responsible to create thread using the arguments provided. It can set its own priority level and other parameters for the thread or use parameters handed over by 'pIntThreadData' argument. After thread is successfully created it should initialize 'pExtThreadData' with the corresponding thread identifier. Please see the definitions above for the exact type of the thread_id field on specific platforms.

Parameters
[in]pIntFuncPointer to the internal function prepared by the framework to be called from the new thread that this function creates
[in]pIntThreadDataPointer to the thread parameters prepared by the framework
[in,out]pExtThreadDataPointer to the structure to be filled with extrenally created thread data. User function must supply identifier for the new thread in the 'thread id' field.
Returns
0 if no errors happened, non zero value otherwise
typedef int(* nuance_common_IThreadAPI_JoinFunc) (nuance_common_IThreadAPI_External *pExtThreadData)

User defined external function that waits for the thread to complete (join).

External function is responsible to cleanup resources allocated during the external create thread call. Thread identifier to be used for wait is provided by the framework in the structure pointed by 'pExtThreadData' argument.

Parameters
[in]pExtThreadDataPointer to the structure containing thread identifier to join.
Returns
0 if no errors happened, non zero value otherwise

Installs external (user defined) functions for thread management.

Install external thread management functions before using the API functions that actually create threads. Once installed it is not recommended to uninstall external thread management functions except as a cleanup sequence after all created threads have been completed and external functions are not needed anymore.

Parameters
[in]instancePointer to the instance of the IThreadAPI object.
[in]customThreadCreateFuncPointer to external thread create function.
[in]customThreadJoinFuncPointer to external function to join the thread.

The result code of the execution
