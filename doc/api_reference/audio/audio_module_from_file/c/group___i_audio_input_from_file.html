typedef struct _nuance_audio_IAudioInputFromFile nuance_audio_IAudioInputFromFile

Interface of the 'AudioInputFromFile' audio module, for DDFW only.

This interface is DEPRECATED and for DDFW only. SDK users shall use the AudioFromFile interface.

In general the AudioInputFromFile module offers the possibility to stream audio data from an audio file list with 1 to n elements to subsequent connected audio modules. The streaming is automatically started and stopped when an audio session becomes active respectively inactive. For each audio session one file is streamed only. Each start of a session triggers the AudioInputFromFile audio module to stream the next audio file from the file list. In case the file list would exceed it is streamed from the beginning again.

The AudioInputFromFile module is usable in two manifestations:

1. AUTOMATICALLY CREATED WITH A PRE-CONFIGURED AUDIO FILE LIST IN THE AUDIO CONFIGURATION FILE (.json): With calling prepareScenario() resp. activateScenario() an involved AudioInputFromFile instance is automatically created. NOTE: In this use case the audio_files parameter in the audio configuration file (.json) is mandatory! With automated creation there is no interface handle available to set a file list by customer API.

2. CREATED BY THE CUSTOMER WITH THE OPTION TO SET AN AUDIO FILE LIST DYNAMICALLY BY THE API: Use the following customer API to create and set up an AudioInputFromFile module. NOTE: In this use case the audio_files parameter in the audio configuration file (.json) is optional in case it is set by the customer API before an audio session is started!

Supported file formats are:

  • standard wav files
  • NIST files
  • header-less raw files, it has to be kept in mind, that multichannel signals are expected in interleaved format (123123123123 and not 111122223333).
  • OGG/OPUS files
  • encoded audio files with a constant bitrate (blocksize needs to be configured as 'samples_per_channel' in the audio format)

The audio file path of each element in the audio file list (pre-configured or set with customer API) is build using the following mechanism:

1. A file with an absolute path remains unchanged. A set root path is not considered. Example:

  • audio_files="/test_path/test_file.wav" ==> "/test_path/test_file.wav"

2. A file with a relative path is prepended with the pre-configured resp. set root path. The resulting path is expected to be in relation to the JSON configuration directory. Examples (JSON config dir = "../config/json"):

  • root_path="" + audio_files="test_file.raw" ==> "../config/json/test_file.raw"
  • root_path="test_path" + audio_files="test_file.raw" ==> "../config/json/test_path/test_file.raw"
  • root_path="test_path" + audio_files="../test_file.raw" ==> "../config/json/test_file.raw"

nuance_common_ResultCode nuance_audio_IAudioInputFromFile_setRootPath ( nuance_audio_IAudioInputFromFile *  instance,
const LH_CHAR *  rootPath 
)

Sets a relative or absolute path that is prepended to audio files having a relative path. Audio files with an absolute path are not affected. A previously set root path even in the audio configuration file (.json) is being overwritten.

NOTE: This is a pure convenience method to prevent the customer from the need to add explicitly a root path in the file list entries in case it is the same. The root path may be empty to 'clear' a previously set one.

A relative path is considered to be in relation to the JSON configuration directory.

The set root path becomes valid with a call of setFile() or setFileList().

An error is returned in case audio file streaming is active when calling this method or when the given root path does not exist.

Parameters
[in,out]instanceThe instance pointer
[in]rootPathRoot path to be prepended to audio files.

Sets an audio file list. A previously set file list even in the audio configuration file (.json) is being overwritten. For a list of supported file formats please have a look into the general description of the AudioInputFromFile module above.

The file list will be iterated in a circular manner each time an audio session is being started (e.g. when a recognition is started). If the end of the list is reached the next capture request will be serviced using the first file of the list again.

NOTE: In case a root path is set it is prepended to audio files with a relative path. Audio files with an absolute path are not affected. It is allowed to mix audio files with absolute and relative path in one list.

An error is returned in case audio file streaming is active or a file of the file list does not exist.

Parameters
[in,out]instanceThe instance pointer
[in]fileListAudio file list to be set.
[in]nfileListNumber of elements in fileList
nuance_common_ResultCode nuance_audio_IAudioInputFromFile_getPadNames ( const nuance_audio_IAudioInputFromFile *  instance,
nuance_common_StringList **  sinkPads,
nuance_common_StringList **  sourcePads 
)

Returns the sink and source pad name(s) of this SDK audio module.

The pad names are given in the configuration of this audio module. In case this audio module does only have one single sink respectively source pad the pad name is equal to the instance name of this module. The pad names have to be unambiguous as they are used as identifier of a specific pad. They are used in conjunction with the instance name to establish/abolish pad connections. The pad name order in the list corresponds the descending order of the sink resp. source pads of this audio module.

This method is thread safe.

Parameters
[in,out]instanceThe instance pointer
[out]sinkPadsSink pad name list of this audio module.
[out]sourcePadsSource pad name list of this audio module.
