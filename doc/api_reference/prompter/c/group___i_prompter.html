typedef struct _nuance_prompter_IPrompter nuance_prompter_IPrompter

This class represents a speech Prompter.

Instances of this class are able to create objects with an IPrompt interface. These represent the prompts during playback and give control over the playback operation.

Background operations initiated with asynchronous method calls are queued in the background and executed according to the order of the method call sequence.

Before destruction of an instance of this class it must be ensured that all its created IPrompt objects are destroyed.

This method is asynchronous and returns instantly after starting the instance creation in the background. Warnings and errors that may occur during creation are reported to the listener nuance_prompter_IPrompterListener. After successful creation, the instance continues to use this listener to report report warnings, errors, or various results or states during subsequent operations.

The method nuance_prompter_IPrompter_waitForCompletion can be utilized to wait for the creation to complete.

nuance_common_ResultCode nuance_prompter_IPrompter_getAvailableSpeechConfigurations ( const nuance_prompter_IPrompter *  instance,
nuance_common_StringList **  languageCodes,
nuance_common_StringList **  voiceNames,
nuance_common_StringList **  operatingPoints,
nuance_common_StringListList **  mlLanguageSets,
nuance_common_StringList **  useCaseNames 
)

Obtains available speech configurations.

The output arguments are (unsorted) lists of strings describing available speech configurations. All lists have the same length and each list represents a column of the speech configuration table. All elements having the same list index describe one speech configuration.

On a "local" type of the IPrompter instance a speech configuration is identified via a set of speech parameters (language code, voice name, voice operating point, multi-lingual language set, e.g.: "enu", "ava-ml", "embedded-high", ["enu", "spm"]) and a usecase name (e.g.: "MyJsonUseCaseName"). The usecase name refers to a usecase JSON configuration located under the Prompter's JSON configuration path. The usecase name is required to activate a usecase on the IPrompter instance.

A speech parameter set describes a certain voice available under the configured "vocalizer_path". A certain voice is considered "available" when its corresponding "broker header file" is found during creation of the IPrompter instance. Important to mention is that there is no detection if the data components (stored in *.dat files) belonging to the voice are incomplete. Thus this method relies on correctly deployed voice data under the given "vocalizer_path".

The returned lists are reduced to names of usecases having a configuration matching to available speech parameters. Contrariwise speech parameters not used in any of the usecase configurations are not listed, too.

The multi-lingual (ML) sets include the voice's native language and optionally the list of languages for which a ML voice can read text using the foreign phoneme set supported by foreign units in its speechbase. This list is read from the noclmset param in the voice's broker header file (*.hdr).

On a "cloud" type of the IPrompter instance a speech configuration looks different. It does not contain information about voice operating point or ML language sets and may contain either a language code or a voice name or both. Typically a speech configuration of the cloud may look like following: "eng-USA", "ava-ml", "", [], "MyJsonCloudUseCase". The returned lists contain only usecases that have at least defined a language or a voice name. All other usecase are considered as invalid and are not activatable. Important to note is that the returned table of speech configurations does not reflect the actual voices and languages available on the cloud service, i.e. the configured speech configurations are not validated against the voices the cloud actually provides. Further note: During the life-time of a IPrompter instance, dynamic modifications of the voice names or language codes with the "vh_callback" mechanism are not reflected in the returned speech configurations.

Returns information about the language with the given languageCode.

This method returns information about the language with the given languageCode. The type of information has to be specified with the infoType argument. The information is returned as a list of strings. The returned content of the string list varies depending on the given infoType.

  • The infoType CLM_VERSION_INFO causes the function to return a list with these strings:

    • list index 0: Version info of the CLM data file.
    • list index 1: Version info of the source language.
    • list index 2: Version info of the target language.

  • The infoType NTS_VERSION_INFO causes the function to return a list with these strings:
    • list index 0: NT-SAMPA version info. Tells the L&H+ version and the NT-SAMPA version.

Returns an error result if the requested information cannot be obtained.

nuance_common_ResultCode nuance_prompter_IPrompter_getConceptString ( const nuance_prompter_IPrompter *  instance,

const LH_CHAR *const *  paramValues,

Obtains a Prompter concept as a string from a concepts database (OTD-file).

Concept database OTD files are only supported by CDFW applications.

The returned concept string can be played as a prompt by passing it to nuance_prompter_IPrompter_playString. The loading of a certain OTD-file is configurable via JSON in the usecase configuration of the IPrompter instance. The un-/loading of the concepts database happens during activation of the corresponding usecase. Therefore it is required to have the corresponding usecase activated before calling this method.

This method allows to pass parameter values as a list of string elements. Note that the mapping of the list elements to the parameters within a concept is done based on the list indices of the elements. It is crucial to fill the parameter value list in the correct order.

If the requested concept is not found or not retrievable of any reason, an error result will be returned.

[in]levelthe user level to filter for concepts categorized into a certain level
[in]randomizeindicates to fetch a concept string randomly if there are alternative concept strings available
[in]paramValueslist containing parameter values required for the requested concept
[in]nparamValuesNumber of elements in paramValues
[out]conceptStringthe returned concept string

)

Sets the given parameter to the given parameterValue.

This method is asynchronous and starts the changing of the parameter as a background operation. This Prompter-related operation can be controlled via the nuance_common_IBackgroundOperation interface the IPrompter instance provides.

Warnings and errors that may occur during this operation are reported to the listener nuance_prompter_IPrompterListener.

This method causes the Prompter to overrule the value of the parameter of the currently activated usecase with the new parameterValue. A subsequent call to nuance_prompter_IPrompter_activateUseCase resets the parameter value according to the value configured in the usecase. As a precondition for this method the IPrompter instance must have a valid usecase activated, otherwise the parameter value is not changed and nuance_prompter_IPrompterListener reports an error.

The method nuance_prompter_IPrompter_waitForCompletion can be utilized to wait for the operation to complete.

The possible value range of parameterValue depends on the given parameter type:

  • VOLUME: (0..100)
  • SPEECH_RATE: (50%..400%)
  • PITCH: (50%..200%, pitch=50%: one octave lower, pitch=200%: one octave higher)
  • TIMBRE: (50%..200%, for values > 100% the voice sounds younger, values < 100% make the voice sound older.)

The method returns an error result if the method is not supported by the underlying TTS engine, e.g. if using a cloud IPrompter instance.

nuance_common_ResultCode nuance_prompter_IPrompter_pause ( nuance_prompter_IPrompter *  instance)

Switches the IPrompter instance to paused state.

During pausing an ongoing play operation is interrupted immediately (synthesis and audio output are stopped). Even if there is no ongoing play operation the IPrompter instance will enter the paused state. All asynchronous operations (e.g. nuance_prompter_IPrompter_playString, nuance_prompter_IPrompter_activateUseCase, ...) are queued but not executed until the IPrompter instance leaves paused state again. The paused state is left with call to nuance_prompter_IPrompter_resume.

Note: A call to nuance_prompter_IPrompter_pause with a subsequent call to nuance_prompter_IPrompt_waitForCompletion within the same application thread causes a deadlock. The recommended way is to make calls to nuance_prompter_IPrompter_pause and nuance_prompter_IPrompter_resume from a separate application thread.

The method returns an error result if the method is not supported by the underlying TTS engine, e.g. if using a cloud IPrompter instance.

Parameters
[in,out]instanceThe instance pointer
Returns
Returns OK if paused successfully, ERROR otherwise. Returns OK if pause() called in already paused state.

Causes the IPrompter instance to leave the paused state.

A previously interrupted play operation is continued again. The instance continues to process queued asynchronous operations (e.g. nuance_prompter_IPrompter_playString, nuance_prompter_IPrompter_activateUseCase, ...).

The method returns an error result if the method is not supported by the underlying TTS engine, e.g. if using a cloud IPrompter instance.

nuance_common_ResultCode nuance_prompter_IPrompter_analyzeText ( nuance_prompter_IPrompter *  instance,
const LH_CHAR *  text 
)

Analyzes the given text.

This method splits the passed UTF8-encoded text block into segments using the underlying Cerence TTS Embedded API ve_ttsAnalyzeText. This may include rewriting of the input text according to loaded user rules. To be able to start text analysis a valid usecase must have been activated on the IPrompter instance.

This method is asynchronous and starts the text analysis as a background operation. This Prompter-related operation can be controlled via the nuance_common_IBackgroundOperation interface the IPrompter instance provides.

During the text analysis operation results and progress information are reported to the listener nuance_prompter_IPrompterListener. Each segmented text unit is sent to the application with nuance_prompter_IPrompterListener_onAnalyzeTextResult. The end of the text analysis operation is signaled with nuance_prompter_IPrompterListener_onAnalyzeTextFinished.

Warnings and errors that may occur during this operation are reported to the listener nuance_prompter_IPrompterListener.

The method returns an error result if the method is not supported by the underlying TTS engine, e.g. if using a cloud IPrompter instance.

Parameters
[in,out]instanceThe instance pointer
[in]textUTF8-encoded text block to analyze.

Returns the sink and source pad name(s) of this SDK audio module.

The pad names are given in the configuration of this audio module. In case this audio module does only have one single sink respectively source pad the pad name is equal to the instance name of this module. The pad names have to be unambiguous as they are used as identifier of a specific pad. They are used in conjunction with the instance name to establish/abolish pad connections. The pad name order in the list corresponds the descending order of the sink resp. source pads of this audio module.
