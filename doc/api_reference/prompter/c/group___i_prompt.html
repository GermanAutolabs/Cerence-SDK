typedef struct _nuance_prompter_IPrompt nuance_prompter_IPrompt

This class represents an interface to a running speech prompt.

Classes implementing this interface are derived from background operations and will enable the user to wait for completion or abort the prompt.

Function Documentation

nuance_common_ResultCode nuance_prompter_IPrompt_skipToMarker ( nuance_prompter_IPrompt *  instance,
LH_S32  position,
nuance_prompter_SkipMode  mode 
)

Restarts the playing prompt at the marker position described by position and mode.

Skipping affects the prompt only if it is playing, means within the time period defined by the nuance_prompter_IPromptListener_onEvent notifications START_PLAY_EVENT and END_PLAY_EVENT.

Supported markers are text units and bookmarks. The type of the text units this function relates to, depends on how the prompter has configured its readmode.

The function allows skipping in the following modes:

  • 1. TEXTUNIT_ABSOLUTE: Skips to the text unit at given position. The value correspondents to the text unit number counted from the beginning of the prompt. The first text unit of the prompt has the position 1.
  • 2. TEXTUNIT_RELATIVE: Skips to the text unit at position counted relatively from the currently playing text unit. The given position defines how many text units to skip and in which direction. The position -1 means to skip one text unit back. The position 1 means to skip one text unit forward. And position 0 means to skip to the beginning of the currently playing text unit.
  • 3. BOOKMARK_ID: Skips to the bookmark at given position, whereas the position contains the bookmark ID used in the text to play.
  • 4. BOOKMARK_RELATIVE: Analogically to the mode TEXTUNIT_RELATIVE it skips to the bookmark at position counted relatively from the currently playing bookmark.

The function returns an error result if marker referred by position does not exist or if the prompt is not playing.

The method returns an error result if the method is not supported by the underlying TTS engine, e.g. if using a cloud IPrompter instance.

nuance_common_ResultCode nuance_prompter_IPrompt_waitForCompletionWithTimeout ( nuance_prompter_IPrompt *  instance,
LH_S32  timeout,
nuance_common_IBackgroundOperation_Status *  status 
)

Waits until the background operation is finished. If the worker thread supports queueing of background operations (e.g. nuance::prompter::IPrompter::activateUseCase() .. nuance::prompter::IPrompter::playString()), this method waits until all operations which are in the queue finish or until timeout expires, providing the timeout value is positive. If timeout value is negative or 0 (undefined) then it waits until all operations finish.

nuance_common_ResultCode nuance_prompter_IPrompt_waitForCompletionAsync ( nuance_prompter_IPrompt *  instance,
nuance_common_IBackgroundListener *  listener 
)

Returns directly and triggers an internal wait for the background operation to finish. If the worker thread supports queueing of background operations (e.g. nuance::prompter::IPrompter::activateUseCase() .. nuance::prompter::IPrompter::playString()), the listener is called once all operations are finished that were in the queue when waitForCompletion() was called.

Note: This method is intended for DragonDriveFramework use only.
